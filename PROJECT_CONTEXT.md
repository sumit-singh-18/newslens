# NewsLens - Project Context

## Project Overview
NewsLens is an AI-powered news bias and perspective analyzer. Users can search any news topic and instantly see:
- How different news outlets cover the same story
- Political bias spectrum across sources
- Sentiment analysis (positive/neutral/negative)
- "Missing Angle" — insights generated by **Google Gemini** (Pro primary, Flash fallback) about what perspectives or facts are missing from the coverage
- Longitudinal tracking: How narratives, bias, and tone evolve over days and weeks

**Core Differentiator**: Unlike one-off chatbot responses, NewsLens stores historical data and enables users to see shifts in media narratives over time.

## Key Objectives
- Pull live articles from multiple outlets using NewsAPI
- Analyze sentiment and bias using HuggingFace models
- Generate intelligent "Missing Angle" using Gemini Pro / Flash
- Store data with proper timestamps for longitudinal analysis
- Build a clean, visual React dashboard with charts and bias spectrum
- Deploy a fully functional web application (Phase 5)
- Later build a React Native mobile version (Phase 6)

## Tech Stack
- **Backend**: Python + FastAPI
- **News Source**: NewsAPI (free tier with caching)
- **NLP**: HuggingFace Transformers (cardiffnlp/twitter-roberta-base-sentiment + bias models)
- **LLM**: Google Gemini (`GEMINI_PRO_MODEL` / `GEMINI_FLASH_MODEL`)
- **Database**: SQLite for development, PostgreSQL for production
- **Frontend**: React + TypeScript + Tailwind CSS + Recharts
- **Testing**: pytest
- **Deployment**: Render (backend) + Vercel (frontend)

## Development Philosophy
- **Backend-First Approach**: The `/analyze?topic=` endpoint must be complete, well-tested, and return rich JSON before any frontend work begins.
- Implement intelligent caching to respect NewsAPI rate limits (check last 24 hours before fetching).
- Design database schema with longitudinal analysis in mind (`snapshot_date`, `fetched_at`, `published_at`).
- Write clean, testable code with proper unit tests.
- UI must be modern, clean, professional, and fully responsive.

## Important Notes
- Never hardcode API keys or secrets. Use environment variables.
- Always implement graceful error handling for external services.
- Focus on code quality — this project will be used in interviews and portfolio.
- Commit after completing each phase.
